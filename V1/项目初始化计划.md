### 2.2 开发步骤安排

#### Step 1：项目初始化（1-2天）

**目标**：建立基础项目结构，可运行的Hello World版本

**任务清单**：
```bash
# 1. 创建项目基础结构
mkdir -p src/main src/renderer src/shared/types src/shared/utils src/python
mkdir -p resources/images resources/icons scripts

# 2. 初始化Node.js项目
npm init -y

# 3. 安装核心依赖
npm install --save-dev electron typescript @types/node @types/react @types/react-dom
npm install --save react react-dom

# 4. 配置TypeScript和Electron
```

**创建的关键文件**：
- `package.json` - 项目配置和依赖
- `tsconfig.json` - TypeScript配置
- `electron-builder.yml` - 打包配置
- `.gitignore` - Git忽略文件
- `README.md` - 项目说明

**验证标准**：
- 可以运行 `npm run dev` 启动开发服务器
- 可以看到Electron窗口显示"Hello World"

#### Step 2：屏幕捕获模块（3-4天）

**目标**：实现轻量级屏幕捕获，捕获关键帧而非连续视频

**技术要点**：
- macOS: 使用 `CGWindowListCreateImage` 和 `Quartz`
- Windows: 使用 GDI API
- Linux: 使用 X11/Wayland API

**创建的文件**：
- `src/main/observation/ScreenCapture.ts`

**功能要求**：
```typescript
interface ScreenCaptureConfig {
  captureInterval: number;  // 捕获间隔（毫秒）
  quality: number;          // 压缩质量 0-1
  maxResolution: string;    // 最大分辨率
}

interface CapturedFrame {
  timestamp: number;
  buffer: Buffer;
  width: number;
  height: number;
  windowTitle?: string;
  application?: string;
}

class ScreenCaptureModule {
  constructor(config: ScreenCaptureConfig);
  start(): Promise<void>;
  stop(): Promise<void>;
  getRecentFrames(limit?: number): CapturedFrame[];
  onFrame(callback: (frame: CapturedFrame) => void): void;
}
```

**验证标准**：
- 能捕获当前屏幕（不记录连续视频，仅关键帧）
- 能识别当前活动窗口标题
- 内存占用合理（< 200MB）
- 性能影响小（CPU < 10%）

#### Step 3：操作记录模块（2-3天）

**目标**：记录系统级操作事件（窗口切换、应用启动等）

**技术要点**：
- 监听系统级事件
- 跨平台兼容性处理

**创建的文件**：
- `src/main/observation/ActionRecorder.ts`

**功能要求**：
```typescript
interface OperationEvent {
  id: string;
  timestamp: number;
  eventType: 'window_switch' | 'app_launch' | 'app_close' | 'file_operation' | 'url_change';
  application: string;
  windowTitle?: string;
  url?: string;
  filePath?: string;
  description: string;
}

class ActionRecorderModule {
  constructor();
  start(): void;
  stop(): void;
  getEvents(startTime?: number, endTime?: number): OperationEvent[];
  onEvent(callback: (event: OperationEvent) => void): void;
}
```

**验证标准**：
- 能正确记录窗口切换
- 能识别应用程序名称
- 能记录文件操作（打开/保存）
- 事件记录不影响系统性能

#### Step 4：本地数据存储（3-4天）

**目标**：实现加密的SQLite数据库存储

**技术要点**：
- SQLite 数据库操作
- AES-256 数据加密
- 数据库迁移和备份

**创建的文件**：
- `src/main/data/Database.ts`
- `src/main/data/Encryption.ts`

**功能要求**：
```typescript
interface DatabaseConfig {
  dbPath: string;
  encryptionKey: string;
}

class Database {
  constructor(config: DatabaseConfig);

  // 操作日志
  insertOperation(event: OperationEvent): Promise<void>;
  getOperations(startTime?: number, endTime?: number): Promise<OperationEvent[]>;

  // 每日摘要
  insertDailySummary(summary: DailySummary): Promise<void>;
  getDailySummary(date: string): Promise<DailySummary | null>;

  // 用户配置
  setConfig(key: string, value: any): Promise<void>;
  getConfig(key: string): Promise<any>;
}
```

**验证标准**：
- 数据能正确加密存储
- 数据库查询性能良好
- 支持数据备份和恢复
- 数据完整性保证

#### Step 5：基础UI界面（3-4天）

**目标**：实现工作时间轴和每日摘要界面

**技术要点**：
- React + TypeScript
- 图表可视化（使用 ECharts 或 Chart.js）
- 响应式设计

**创建的文件**：
- `src/renderer/components/WorkTimeline.tsx`
- `src/renderer/components/DailyReport.tsx`
- `src/renderer/App.tsx`

**验证标准**：
- 工作时间轴能正确显示
- 每日摘要能自动生成
- UI响应流畅，无明显卡顿

#### Step 6：集成与测试（2-3天）

**目标**：整体集成，功能测试，性能优化

**任务清单**：
- 各模块集成
- 端到端测试
- 性能优化
- Bug修复

**验证标准**：
- 所有功能正常工作
- 稳定性测试通过（24小时连续运行）
- 内存和CPU使用在合理范围
- 用户体验流畅

### 5.2 步骤2：创建基础文件
```bash
# 创建基础目录结构
mkdir -p src/main src/renderer src/shared/types

# 创建 TypeScript 配置
cat > tsconfig.json << 'EOF'
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "jsx": "react-jsx"
  },
  "include": ["src/**/*"]
}
EOF

# 创建 Vite 配置
cat > vite.config.ts << 'EOF'
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  base: './',
  server: {
    port: 5173
  }
});
EOF

# 创建主进程入口
cat > src/main/index.ts << 'EOF'
import { app, BrowserWindow } from 'electron';
import path from 'path';

function createWindow() {
  const win = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      nodeIntegration: true,
      contextIsolation: false
    }
  });

  if (process.env.NODE_ENV === 'development') {
    win.loadURL('http://localhost:5173');
    win.webContents.openDevTools();
  } else {
    win.loadFile(path.join(__dirname, '../renderer/index.html'));
  }
}

app.whenReady().then(createWindow);

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit();
  }
});

app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) {
    createWindow();
  }
});
EOF

# 创建渲染进程入口
cat > src/renderer/index.html << 'EOF'
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Avatar Project</title>
</head>
<body>
  <div id="root"></div>
  <script type="module" src="./index.tsx"></script>
</body>
</html>
EOF

cat > src/renderer/index.tsx << 'EOF'
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
EOF

cat > src/renderer/App.tsx << 'EOF'
import React from 'react';

const App: React.FC = () => {
  return (
    <div style={{ padding: '20px' }}>
      <h1>Avatar Project</h1>
      <p>欢迎使用你的数字工作伙伴！</p>
    </div>
  );
};

export default App;
EOF
```
